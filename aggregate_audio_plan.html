Aggregate EgoCom Inference Audio

This document describes the implementation added in /home/prj/comp_utils/aggregate_audio.py and /home/prj/comp_utils/aggregate_audio.sh.

Manifest /home/prj/data/egocom_holdout/4s_overlap0_v2_day1_con4_parts/test/manifest/manifest_mm.jsonl Audio root /home/prj/ego2ego_mag/result/inference-geo-label/audio Output root /home/prj/ego2ego_mag/result/inference-geo-label/aggregated_audio

Implementation

  • Read the JSONL manifest and preserve each row index.
  • Parse inference sample directories named like sample_0000_day_1__con_4__part1.
  • Map each sample number to a manifest row, with sample_0000 mapping to row 0.
  • Group mapped rows by scene_name and actual person perspective by default, producing 12 directories for this manifest.
  • Build src_audio.wav from rows where the person is src_person.
  • Build tgt_audio.wav and pred_audio.wav from rows where the person is tgt_person.
  • De-duplicate all three audio names by clip identity, keeping the earliest manifest row when multiple source persons predict the same target clip.
  • Optional legacy grouping remains available with --group-by scene-src, --group-by scene-src-tgt, or --group-by scene.
  • Sort each group by clip_start_ms, clip_end_ms, and manifest row index.
  • For every group, concatenate pred_audio.wav, src_audio.wav, and tgt_audio.wav separately.
  • Use ffmpeg's concat demuxer with stream copy, so compatible WAV segment data is not re-encoded.
  • Write summary.json with group metadata, output paths, skipped groups, and errors.

Timing Policy

The selected policy is strict concatenation. The script appends segments in manifest time order and does not insert silence, trim overlaps, or crossfade. This matches the current 4s_overlap0 manifest.

Output Layout

Each scenario/source-person pair gets one output directory:

aggregated_audio/
  day_1__con_4__part1__person_1/
    pred_audio.wav
    src_audio.wav
    tgt_audio.wav
  summary.json

Usage

cd /home/prj/comp_utils
./aggregate_audio.sh --dry-run
./aggregate_audio.sh --overwrite
./aggregate_audio.sh --group-by scene --output-root /path/to/aggregated_audio_scene_dedup --overwrite
./aggregate_audio.sh --group-by scene-src-tgt --overwrite

Override paths with environment variables or CLI flags:

MANIFEST=/path/to/manifest.jsonl \
AUDIO_ROOT=/path/to/inference/audio \
OUTPUT_ROOT=/path/to/aggregated_audio \
./aggregate_audio.sh --strict --overwrite

Validation

  • --dry-run prints all planned groups and outputs without creating WAV files.
  • --strict exits non-zero if any manifest row lacks a sample directory, any sample maps outside the manifest, or a group is missing audio files.
  • Without --overwrite, existing outputs are skipped.
  • Use ffprobe on generated WAV files to confirm codec, sample rate, channel count, and duration.
