Pair-Level EgoCom Audio Aggregation Plan

Fix aggregation so each scene keeps directed transfer pairs separate, such as person_1->person_3 and person_2->person_3, producing 24 aggregate outputs for the 10s holdout manifest.

Summary Diagnosis Implementation Timeline Risk Table Validation

Summary

Correct grouping
scene_name + src_person + tgt_person
Expected count
4 scenes x 6 directed pairs = 24 aggregate directories
Output naming
day_1__con_4__part1__person_1__person_3

The previous default grouped by target/source person timeline and merged predictions from different source people into one target-person output. The corrected default keeps each directed transfer pair independent.

Diagnosis

HighThe old default scene-person created only 12 aggregate outputs because each target person was represented once per scene.

MediumThe code already had a pair-level mode named scene-src-tgt, and a dry run against the 10s manifest produced 24 groups. The main defect was the default behavior and stale output handling.

10s manifest rows: 487
Pair-level groups: 24
Per scene: 6 directed pairs
Example target person_3 outputs:
  day_1__con_4__part1__person_1__person_3
  day_1__con_4__part1__person_2__person_3

Implementation

  • Change the default --group-by value in aggregate_audio.py to scene-src-tgt.
  • Update the module docstring and CLI help to describe directed source-target pair aggregation.
  • Keep scene-person as an explicit legacy option for old workflows.
  • Add --clean-output-root to remove existing child directories from aggregated_audio before regeneration.
  • Record clean_output_root and cleaned_output_dirs in summary.json for auditability.
