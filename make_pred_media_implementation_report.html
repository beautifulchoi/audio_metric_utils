make_pred_media Implementation Report

Project-local report for the /home/prj/comp_utils changes. The generator now uses inference summary.json, resolves trimmed videos from the new holdout data root, and writes camera + moving comparison-plot composites.

Summary Files Pipeline Diagram Progress Behavior Validation Risks

Summary

Implemented Each sample is mapped through <infer-root>/summary.json first. Source and target camera clips are resolved from /home/prj/data/egocom_holdout/4s_overlap0_v2_day1_con4_parts/test/video.

Latest change The animated side plot is no longer a single-audio plot. Every pred, target, and source output now shows Source, Target, and Predicted waveform rows together, with L/R spectrograms and a moving time bar across all rows.

Audio flip The wrapper defaults to no audio flip because the new data is already flipped. Callers can still opt in with AUDIO_FLIPPED_ARGS=--audio-flipped for old data.

Inputs

  • summary.json is the primary manifest.
  • src_audio.wav, tgt_audio.wav, and pred_audio.wav are loaded for every plot panel.
  • Source and target videos come from the new data-root video/ directory.

Outputs

  • Default output root is <infer-root>/pred_media.
  • Sample layout writes pred.mp4, target.mp4, and source.mp4.
  • Each MP4 is camera video stacked beside the moving all-audio comparison plot.

Progress Details

1. Inspected old reference outputs

Checked prior pred_media_samples and plot_media_summary.json. The reference output shape was side-by-side camera plus animated plot media.

2. Switched to summary-driven mapping

Added logic to read summary.json first, resolve explicit paths when present, and fall back only for older inference folders.

3. Restored composite media

Changed output generation from camera-only muxing to make_composite_media, which renders a temporary plot video and stacks it beside the camera clip.

4. Added all-audio plot panels

Updated the plot renderer so every output loads source, target, and pred audio together. The animated plot now contains three waveform rows and paired L/R spectrograms instead of only the selected output's waveform.

5. Updated wrapper defaults

The wrapper now defaults to no audio flip and exposes rendering controls for plot FPS, output FPS, DPI, and camera scale.

6. Saved smoke output under the project

Ran one-sample validation into /home/prj/comp_utils/make_pred_media_smoke_all_waveforms.
